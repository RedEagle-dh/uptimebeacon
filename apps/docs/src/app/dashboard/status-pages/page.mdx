# Status Pages

Create beautiful, branded status pages to keep your users informed about service availability. Status pages automatically update when incidents occur and provide transparency about your infrastructure health.

## ğŸ¯ What is a Status Page?

A status page is a public-facing webpage that displays the real-time status of your services. It helps you:

- **Build Trust**: Show transparency about service health
- **Reduce Support Load**: Users check status before contacting support
- **Professional Image**: Demonstrate reliability and openness
- **Keep Users Informed**: Automatic updates during incidents

## ğŸŒ Custom Domain Setup

### DNS Configuration

**Step 1: Add CNAME Record**
```
Type: CNAME
Name: status
Value: example.uptimebeacon.com
TTL: 3600
```

**Step 2: Verify in Dashboard**
1. Go to Status Page settings
2. Enter custom domain: `status.example.com`
3. Click "Verify DNS"
4. Wait for propagation (5-60 minutes)

**Step 3: SSL Certificate**
- Automatic via Let's Encrypt
- Usually takes 2-5 minutes
- Renews automatically every 90 days

### Troubleshooting DNS

**DNS not propagating?**
```bash
# Check DNS resolution
dig status.example.com CNAME

# Check from multiple locations
nslookup status.example.com 8.8.8.8
```

**SSL certificate issues?**
- Ensure DNS is fully propagated
- Check CAA records don't block Let's Encrypt
- Wait 24 hours for full propagation
